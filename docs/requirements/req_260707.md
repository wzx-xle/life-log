## 一、产品概述

### 1.1 产品定位

LifeLog 是一款基于地理位置的本地生活服务体验记录工具，以 PWA 形态运行于移动端浏览器。用户可在消费后记录对线下服务场所（餐厅、酒店、零售店等）的体验评价，并通过地图浏览、分类筛选、统计分析等方式回顾个人消费足迹。

### 1.2 核心特性

- **纯本地优先**：所有数据存储于浏览器本地 IndexedDB，无服务端依赖
- **地图驱动**：以地图为主视图，直观展示服务商分布与个人足迹
- **离线可用**：Service Worker 缓存静态资源，离线状态下完整可用
- **隐私保护**：4 位数字密码锁，无账号体系，数据完全由用户掌控
- **PWA 安装**：可添加至设备主屏幕，获得类原生应用体验

### 1.3 用户画像

- 习惯记录消费体验、撰写评价的生活记录者
- 关注周边生活服务、希望积累个人消费参考信息的用户
- 对数据隐私敏感、不希望数据上云的群体

### 1.4 设备与平台

- 运行环境：移动端浏览器（iOS Safari / Android Chrome）
- 适配方式：PWA，可安装到主屏幕
- 屏幕方向：竖屏为主
- 网络需求：首次加载需联网，后续离线可用；地图瓦片需按需联网缓存

### 2.2 详细功能说明

#### 2.2.1 密码保护

**功能概述**

为应用提供 4 位数字密码保护，防止他人未经授权查看用户的消费记录和隐私数据。所有密码逻辑在本地完成，不涉及网络请求。

**密码创建流程**

1. 用户在设置页面开启"密码锁"开关
2. 进入密码设置页，通过数字键盘输入 4 位数字
3. 系统提示"请再次输入以确认"，用户再次输入相同密码
4. 两次输入一致则密码创建成功，不一致则清空输入并提示"两次密码不一致，请重新设置"
5. 密码以 SHA-256 哈希值形式存入 localStorage，不存储明文

**密码验证规则**

- 应用启动时：若密码锁已开启，首先展示密码输入页面
- 后台切回时：根据用户设定的自动锁定时间，判断是否需要重新验证
- 验证方式：用户输入 4 位数字后自动与存储的哈希值比对
- 错误处理：连续输错 5 次后，锁定 30 秒，期间数字键盘完全禁用，页面显示倒计时
- 倒计时结束后错误计数自动清零，用户可重新输入

**密码修改流程**

1. 用户在设置中点击"修改密码"
2. 首先输入当前密码进行身份验证
3. 验证通过后，进入新密码设置流程（输入新密码 → 确认新密码）
4. 两次一致则修改成功，更新存储的密码哈希值

**自动锁定时间配置**

- 可选值：立即、1 分钟、5 分钟、15 分钟
- 逻辑：应用进入后台（页面不可见）时记录时间戳，再次回到前台时计算间隔，超过设定值则展示密码输入页
- 默认值：5 分钟

**密码重置机制**

- 不提供"忘记密码"功能
- 唯一重置方式：在设置中选择"重置密码"，系统弹出警告提示"此操作将清除所有本地数据（包括体验记录、店铺信息等），且不可恢复"
- 用户确认后，清空 localStorage 中密码相关数据以及 IndexedDB 中所有业务数据，应用恢复至首次使用状态

#### 2.2.2 地图主页

**功能概述**

地图主页是应用的核心视图，以地图为载体展示用户已添加的所有服务商位置，提供定位、搜索、快速添加等功能。

**地图展示**

- 使用高德地图作为底图，默认视图模式为 2D
- 首次打开时请求用户授权地理位置权限，获取当前位置并自动居中
- 若用户拒绝授权，默认显示北京中心点，用户可手动搜索或拖动地图
- 支持手势操作：双指缩放、单指平移

**店铺标记**

- 已添加的所有店铺以彩色圆形标记展示在地图上
- 标记颜色按分类区分，方案如下：

| 分类 | 标记颜色 | 色值 |
|------|----------|------|
| 餐饮 | 橙色 | #FF6B35 |
| 住宿 | 蓝色 | #4A90D9 |
| 零售 | 绿色 | #52C41A |
| 生活服务 | 紫色 | #8B5CF6 |
| 娱乐休闲 | 黄色 | #F5A623 |
| 自定义 | 灰色 | #9E9E9E |

- 标记中心显示分类对应的小图标

**标记交互**

- 点击标记：弹出店铺信息卡片，显示店铺名称、综合评分（该店铺所有记录的评分均值）、分类标签
- 卡片提供两个快捷按钮："写记录"和"查看详情"
- 点击卡片外部区域或关闭按钮收起卡片
- 地图空白区域长按：弹出操作菜单"在此位置添加新店铺"

**当前位置定位**

- 地图左下角提供定位按钮
- 点击后地图动画移动至用户当前 GPS 位置
- 定位成功后以蓝色圆点标记当前位置

**搜索功能**

- 页面顶部设置搜索栏，占位提示文字"搜索店铺或地址"
- 输入关键词后调用高德 POI 搜索接口，展示下拉建议列表
- 选中某条建议后，地图定位至对应坐标并临时高亮该位置

**快速添加店铺**

- 底部导航栏中央提供"＋添加"按钮
- 点击后进入添加店铺页面

#### 2.2.3 店铺管理

**添加店铺**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 店铺名称 | 文本输入 | 是 | 最大长度 50 字符 |
| 分类 | 单选 | 是 | 餐饮/住宿/零售/生活服务/娱乐休闲/自定义 |
| 自定义分类名 | 文本输入 | 条件必填 | 当分类选择"自定义"时必填 |
| 地址 | 文本 + 地图选点 | 是 | 可输入文字地址，也可点击跳转地图选点页面 |
| 联系电话 | 文本输入 | 否 | 支持座机和手机号格式 |
| 营业时间 | 文本输入 | 否 | 自由文本，如"09:00-22:00" |
| 标签 | 标签数组 | 否 | 支持自定义添加，如"深夜营业""宠物友好" |
| 照片 | 图片数组 | 否 | 支持拍照或从相册选择，最多 9 张，自动压缩后以 base64 存储 |

- 在店铺详情页点击"编辑"按钮进入编辑模式
- 表单复用添加店铺的字段结构，回填已有数据
- 修改保存后更新 updatedAt 时间戳
- 分类变更时，地图上该店铺的标记颜色同步更新

**删除店铺**

- 在店铺详情页或编辑页提供删除入口
- 删除店铺时，该店铺下的所有关联体验记录一并删除
- 操作前弹出二次确认对话框，明确告知将删除的记录数量

| 零售 | retail | 便利店、超市、商场、专卖店、杂货铺 |
| 生活服务 | service | 理发店、美甲店、健身房、洗衣店、维修店、宠物店 |
| 娱乐休闲 | entertainment | 电影院、KTV、密室逃脱、剧本杀、游乐场 |
| 自定义 | custom | 用户自由命名的类别 |

#### 2.2.4 体验记录

**功能概述**

提供快速和详细两种记录模式。用户可在设置中指定默认模式，也可在添加记录时临时切换。

**快速记录模式字段**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 关联店铺 | 下拉选择或新建 | 是 | 从已有店铺列表中选择，或快速新建店铺 |
| 总体评分 | 星级 1-5 | 是 | 单维度综合评分 |
| 简短感受 | 多行文本 | 否 | 最大 500 字符 |
| 消费金额 | 数字输入 | 否 | 单位：元，保留两位小数 |
| 照片 | 图片数组 | 否 | 最多 9 张，自动压缩 |

**详细记录模式字段**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 关联店铺 | 下拉选择或新建 | 是 | 同快速模式 |
| 消费日期 | 日期选择 | 是 | 默认当天，可修改为历史日期 |
| 服务态度评分 | 星级 1-5 | 否 | 单独维度 |
| 环境卫生评分 | 星级 1-5 | 否 | 单独维度 |
| 性价比评分 | 星级 1-5 | 否 | 单独维度 |
| 综合推荐评分 | 星级 1-5 | 否 | 单独维度 |
| 详细体验 | 多行文本 | 否 | 最大 2000 字符 |
| 消费金额 | 数字输入 | 否 | 单位：元 |
| 消费项目 | 动态列表 | 否 | 每项包含"项目名称"和"价格"，可增删 |
| 个人标签 | 标签数组 | 否 | 如"值得再去""性价比高""环境好" |
| 是否会再光顾 | 单选 | 否 | 选项：会 / 不会 / 不确定 |
| 照片 | 图片数组 | 否 | 最多 9 张 |

**评分交互**

- 星级评分采用点击选中的交互方式
- 点击第 N 颗星则选中 1 到 N 颗星
- 点击已选中的最高星可取消评分（回退到 0 星）
- 已选星星为实心高亮色，未选为空心灰色

**编辑记录**

- 在记录列表或店铺详情中可进入编辑模式
- 表单回填已有数据，修改后保存

**删除记录**

- 列表项左滑展示删除按钮
- 或在记录详情/编辑页提供删除入口
- 删除前弹出确认提示

#### 2.2.5 记录浏览

**列表视图**

- 以时间分组（今天、昨天、本周、更早），组内按日期降序排列
- 每条记录卡片展示：店铺名称、分类色标圆点、总体评分、内容摘要（截取前 50 字）、消费金额、个人标签
- 无记录时展示空状态引导："还没有体验记录，去地图上添加吧"

**分类筛选**

- 列表顶部设置横向可滚动的分类标签栏
- 标签：全部、餐饮、住宿、零售、生活服务、娱乐、自定义
- 选中标签高亮，列表实时筛选对应分类的记录
- 标签栏在滚动时保持吸顶

**排序切换**

- 排序方式：按时间（默认）、按评分、按距离
- 点击排序按钮切换，当前排序方式高亮显示

**关键词搜索**

- 列表顶部搜索栏支持按店铺名称、记录内容关键词搜索
- 输入即时过滤，搜索结果按时间降序展示
- 无匹配结果时提示"未找到相关记录"

#### 2.2.6 店铺详情

**页面结构**

1. 顶部：店铺照片轮播区域（无照片时展示分类默认占位图）
2. 综合评分区：展示该店铺所有记录的综合推荐评分均值，以及记录总数
3. 基本信息区：地址、电话（可点击拨打）、营业时间、标签
4. 操作按钮区：导航按钮（跳转地图 App）、拨打按钮
5. 体验记录区：展示该店铺下所有记录，按时间降序，每条可点击查看详情
6. 底部固定操作栏：编辑店铺、写记录

**数据联动**

- 该店铺下所有记录的评分变动后，详情页综合评分实时更新
- 删除某条记录后，记录数和综合评分同步刷新

#### 2.2.7 统计看板

**功能概述**

以图表形式展示用户的消费统计和偏好分析，帮助用户了解自己的消费习惯。

**统计模块**

| 模块 | 展示形式 | 说明 |
|------|----------|------|
| 概览数据 | 数字卡片 | 总记录数、记录店铺数、总消费金额 |
| 消费趋势 | 折线图 | 按月展示近 12 个月的消费金额变化 |
| 分类消费占比 | 饼图 | 按消费金额统计各分类占比 |
| 评分分布 | 柱状图 | 综合评分 1-5 分的记录数量分布 |
| 常去店铺 TOP5 | 排行列表 | 按记录次数降序排列 |
| 足迹地图入口 | 按钮 | 点击跳转地图页，仅显示有记录的店铺 |

**图表交互**

- 饼图扇区点击可下钻查看该分类的具体店铺
- 折线图支持长按查看某月具体金额
- 排行列表项点击可跳转对应店铺详情

**数据更新**

- 统计页面每次进入时重新计算最新数据
- 新增或删除记录后，统计数据同步更新

#### 2.2.8 数据管理

**数据导出**

- 在设置中点击"导出数据"
- 将所有店铺和记录数据序列化为 JSON 格式，包含版本号和导出日期
- 触发浏览器下载，文件命名格式：lifelog_backup_YYYY-MM-DD.json
- 导出内容中，记录数据补充关联店铺名称，提升可读性

**数据导入**

- 在设置中点击"导入数据"
- 选择 .json 文件后，系统校验文件格式和版本兼容性
- 校验通过后提示选择导入模式：
  - 合并（Merge）：保留现有数据，追加导入数据
  - 覆盖（Overwrite）：清空现有数据，替换为导入数据
- 导入完成后展示导入结果摘要（导入店铺数、记录数）

**清除所有数据**

- 在设置中点击"清除所有数据"
- 弹出警告："此操作将删除所有体验记录和店铺数据，且不可恢复。是否继续？"
- 用户确认后清空 IndexedDB 中所有业务数据
- localStorage 中除密码相关外的设置项一并清除
- 清除完成后返回地图主页，展示空状态

#### 2.2.9 设置中心

**安全设置**

| 设置项 | 类型 | 说明 |
|--------|------|------|
| 密码锁 | 开关 | 开启/关闭密码保护功能 |
| 修改密码 | 导航项 | 进入密码修改流程 |
| 自动锁定时间 | 选择项 | 立即 / 1分钟 / 5分钟 / 15分钟 |
| 重置密码 | 导航项 | 清除所有数据并重置密码 |

**偏好设置**

| 设置项 | 类型 | 说明 |
|--------|------|------|
| 默认记录模式 | 单选 | 快速模式 / 详细模式，影响添加记录时的默认表单 |
| 地图默认缩放级别 | 选择项 | 影响地图主页初始展示范围 |

**关于**

- 应用名称和版本号
- 技术说明：纯本地存储，数据不上传

## 五、核心模块设计

### 5.1 密码锁模块

**状态定义**

密码锁模块管理以下状态：

- 是否开启密码锁（读取 localStorage）
- 当前锁定状态：未锁定 / 密码输入中 / 输错锁定中
- 错误次数计数（存储在 sessionStorage，关闭浏览器后重置）
- 锁定倒计时剩余秒数（30 秒锁定期间）
- 解锁有效时间戳（用于后台切回时判断是否超时）

**状态流转**

1. 应用启动 → 检查密码锁是否开启
2. 未开启 → 直接进入应用主页
3. 已开启 → 进入"密码输入中"状态
4. 输入正确密码 → 解锁，进入应用
5. 输入错误 → 错误计数 +1
6. 错误计数 < 5 → 清空输入框，可重新输入
7. 错误计数 = 5 → 进入"输错锁定中"状态，启动 30 秒倒计时
8. 倒计时结束 → 错误计数清零，回到"密码输入中"状态
9. 应用进入后台 → 记录时间戳
10. 应用回到前台 → 对比时间戳与自动锁定时间，超时则重新进入"密码输入中"状态

**密码存储方案**

- 密码不存明文
- 存储方式：SHA-256(固定盐值 + 原始密码)
- 盐值为应用内硬编码的常量字符串
- 验证时对输入密码执行相同哈希运算后比对

**重置密码方案**

- 唯一途径：清空所有数据
- 执行操作：删除 localStorage 中的 pwd_hash、pwd_enabled 等密码相关键值；清空 IndexedDB 中 places 表和 reviews 表的所有数据
- 执行后应用恢复首次使用状态

**压缩策略**

- 使用 Compressor.js 在客户端对图片进行压缩
- 压缩参数：质量 60%，最大宽高 800px
- 压缩后单张图片 base64 大小控制在 200KB 以内
- 压缩后的图片转为 base64 字符串存储到 IndexedDB

- 照片以 base64 字符串数组形式存储在对应记录或店铺中

### 5.5 数据导入导出模块

**导出流程**

1. 从 IndexedDB 读取所有 places 和 reviews 数据
2. 构建 placeId 到店铺名称的映射表
3. 为每条 review 补充 placeName 字段
4. 组装导出数据结构：版本号、导出日期、places 数组、reviews 数组
5. 序列化为 JSON 字符串
6. 创建 Blob 对象，触发浏览器下载

**导入流程**

1. 用户选择本地 .json 文件
2. 读取文件内容并解析 JSON
3. 校验格式：检查 version 字段是否存在，places 和 reviews 是否为数组
4. 校验通过后提示用户选择导入模式（合并 / 覆盖）
5. 覆盖模式：先清空两张表所有数据
6. 合并模式：逐条导入，使用 id 自动生成，避免主键冲突
7. 导入店铺数据，记录新旧 ID 映射关系
8. 根据 ID 映射更新记录数据的 placeId 后导入
9. 完成后提示导入结果

**注意事项**

- 导入数据中的图片 base64 直接写入，不重新压缩
- 导入后地图和列表需刷新以展示新数据

## 六、PWA 配置

### 6.1 Service Worker 策略

**预缓存（Precache）**

构建时自动预缓存以下静态资源：
- HTML 入口文件
- 编译后的 JS 和 CSS 文件
- 应用图标（favicon、PWA 图标）
- 字体文件

**运行时缓存（Runtime Caching）**

| 资源类型 | 缓存策略 | 缓存名称 | 最大条目 | 过期时间 |
|----------|----------|----------|----------|----------|
| 高德地图 API 请求 | Network First | amap-api-cache | 100 | 24 小时 |
| 地图瓦片图片 | Network First | amap-tile-cache | 500 | 7 天 |
| 其他第三方资源 | Stale While Revalidate | third-party-cache | 50 | 24 小时 |

**缓存策略说明**

- **Network First**：优先请求网络，网络不可用时使用缓存。适用于地图 API 和瓦片，确保尽可能获取最新数据
- **Stale While Revalidate**：优先返回缓存，同时后台更新缓存。适用于不常变化的第三方资源
- **Precache**：构建时缓存，适用于应用自身的静态资源

### 6.3 离线能力

- 静态资源通过预缓存确保离线可访问
- 地图瓦片通过运行时缓存，已浏览区域离线时可查看
- 业务数据存储于 IndexedDB，完全离线可用
- 离线状态下可正常浏览已有数据和添加新记录
- 离线添加的记录直接写入本地 IndexedDB，由于无服务端同步需求，不存在数据冲突问题

### 6.4 更新机制

- 使用 autoUpdate 模式
- Service Worker 检测到新版本后自动更新
- 更新完成后提示用户刷新页面以应用新版本

## 七、非功能需求

### 7.1 性能要求

- 首屏加载时间（含地图初始化）：< 3 秒（4G 网络）
- 页面切换响应时间：< 300 毫秒
- 图片压缩处理时间：单张 < 1 秒
- 列表滚动帧率：保持 60fps，无卡顿
- IndexedDB 查询响应：< 100 毫秒（1000 条记录以内）

### 7.2 兼容性要求

| 平台 | 浏览器 | 最低版本 |
|------|--------|----------|
| iOS | Safari | 15.0+ |
| Android | Chrome | 90+ |
| Android | 系统 WebView | 90+ |
| 桌面（辅助调试） | Chrome / Edge | 90+ |

### 7.3 安全要求

- 密码使用 SHA-256 加盐哈希存储，不存明文
- 所有数据存储于本地浏览器，不向任何服务器发送
- 地图 SDK 请求高德服务器，仅用于地图展示和搜索功能
- 应用不收集任何用户个人信息
- HTTPS 部署为 PWA 运行的必要条件

### 7.4 存储限制

- 单张照片压缩后不超过 200KB
- 单条记录/店铺照片总大小建议不超过 2MB
- IndexedDB 总存储量取决于浏览器限制（通常 50MB-无限制）
- 设置中提供数据导出功能，方便用户定期备份释放空间
